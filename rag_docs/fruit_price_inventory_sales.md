# 사과 가격정보·재고현황·판매현황 더미 데이터

이 문서는 MVP 테스트용 내부 운영 데이터다. 수치는 로봇팔 수확 이벤트 연동 전까지 사용하는 더미 기준값이며, 실제 운영에서는 MariaDB 재고 테이블, 로봇팔 수확 이벤트, 쇼핑몰 주문 데이터로 갱신한다.

## 기준일

- 기준일: 2026-06-08
- 사업장: Manage Apple 테스트 물류센터
- 품목: 사과
- 가격 단위: kg당 원
- 재고 단위: kg
- 크기 등급: 대과, 중과
- 상품성 등급: 상, 중, 하
- 추정 단위중량: 대과 0.32kg/개, 중과 0.24kg/개
- 중량 출처: 실제 계량값이 아니라 크기 등급 기반 추정값

## 로봇팔 수확 이벤트 반영 기준

로봇팔 코드가 수확 성공 이벤트를 보내면 크기 등급과 상품성 등급에 맞춰 재고를 증가시킨다. 무게를 실제로 측정하지 않으므로 크기 등급별 추정 단위중량을 사용한다.

| 수확 결과 | 재고 반영 방식 |
| --- | --- |
| 대과 1개 수확 | 해당 상품성 등급의 기준 가용 재고에 0.32kg 추가 |
| 중과 1개 수확 | 해당 상품성 등급의 기준 가용 재고에 0.24kg 추가 |
| 대과 10개 수확 | 해당 상품성 등급의 기준 가용 재고에 3.2kg 추가 |
| 중과 10개 수확 | 해당 상품성 등급의 기준 가용 재고에 2.4kg 추가 |

## 운영 판단 기준

- 재고 질문에는 크기 등급, 상품성 등급, 기준 가용 재고, 쇼핑몰 등록 재고, 쇼핑몰 추가 등록 가능 재고를 함께 확인한다.
- 쇼핑몰에 상품을 올릴 때는 쇼핑몰 추가 등록 가능 재고를 넘기지 않는다.
- 주문이 들어오면 쇼핑몰 남은 판매수량이 줄어든다.
- 로봇팔 수확 이벤트가 들어오면 크기 등급별 추정 단위중량을 기준으로 재고가 증가한다.
- 하 등급은 일반 판매보다 할인, 가공, 특가 채널을 우선 검토한다.
- 상 등급은 내부 온라인몰의 일반 판매 또는 선물세트 구성에 우선 사용한다.

## 챗봇 답변 예시 규칙

- "대과 상 얼마야?"라고 물으면 대과 상 등급의 kg당 판매가 9,200원을 우선 답한다.
- "중과 하 재고 알려줘"라고 물으면 기준 가용 재고 320kg, 쇼핑몰 등록 재고 20kg, 추가 등록 가능 재고 300kg을 답한다.
- "중과 하 400kg 올려줘"라고 요청하면 추가 등록 가능 재고 300kg을 초과하므로 등록할 수 없다고 안내한다.
- "로봇팔이 중과 상 3개 수확했어"라는 이벤트는 중과 상 기준 가용 재고에 0.72kg을 추가하는 것으로 해석한다.
